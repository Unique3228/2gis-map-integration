# Интеграция с 2GIS API (выбор адреса на карте)

## 📌 Описание проекта

Данный репозиторий содержит OpenAPI-спецификацию и документацию интеграции с сервисом 2GIS, реализованной для отображения интерактивной карты и выбора адреса пользователем на веб-платформе компании

Интеграция позволила реализовать функциональность поиска и выбора адреса через карту с последующей передачей структурированных данных в систему

---

## 🎯 Бизнес-цель

Цель интеграции - улучшение пользовательского опыта при вводе адреса за счет:
- снижения количества ошибок при ручном вводе
- ускорения процесса заполнения адресных данных
- повышения точности геокодирования
- стандартизации адресных данных в системе

## 🔌 Архитектура интеграции

Интеграция реализована по модели клиент–сервер с использованием внешнего геосервиса 2GIS

Основной поток:

1. Пользователь открывает карту на сайте
2. Пользователь выполняет поиск адреса или выбирает точку на карте
3. Фронтенд отправляет запрос в backend сервиса для поиска/геокодирования адреса
4. Backend обращается к внешнему API 2GIS для получения геоданных
5. 2GIS возвращает координаты и структурированный адрес в backend
6. Backend нормализует и возвращает данные на фронтенд
7. Фронтенд отображает результат пользователю

## 📄 API спецификация

В репозитории представлена OpenAPI 3.0 спецификация интеграции с геосервисом 2GIS, включающая набор backend-методов для авторизации, геокодирования и отображения карты

Спецификация описывает следующие функциональные области:

- **Авторизация пользователя**
  - `/auth/login`
  - получение access token для доступа к API
  - управление временем жизни токенов и механизмом рефреша

- **Поиск объектов и подсказки (Suggest API)**
  - `/api/suggest/suggests`
  - формирование подсказок при вводе адреса пользователем
  - поддержка фильтрации по региону, типам объектов и геолокации
  - использование контекста пользователя для ранжирования результатов

- **Геокодирование (Geocoder API)**
  - `/api/geocode/items/geocode`
  - прямое геокодирование (адрес → координаты)
  - обратное геокодирование (координаты → адрес)
  - поддержка параметров фильтрации, сортировки и радиуса поиска

- **Отображение карты**
  - `/api/2gis/tiles`
  - получение тайлов карты в формате изображения (JPEG)
  - работа с координатами тайлов и уровнями масштабирования

Спецификация включает:
- описание REST-эндпоинтов
- параметры запросов (query parameters)
- форматы ответов (JSON, image/jpeg)
- обработку стандартных HTTP-ошибок (400, 403, 404, 408, 500)
- примеры структур ответов для ключевых методов

Документация использовалась для:
- согласования API-контрактов между frontend и backend
- реализации интеграции с внешним сервисом 2GIS
- тестирования и валидации корректности обмена данными
- генерации Swagger-документации
